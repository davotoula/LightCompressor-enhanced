# Release v1.9.0 (library) + app-v1.4.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship minor-version releases of both the LightCompressor library (`v1.9.0`) and the sample app (`app-v1.4.0`) back-to-back: JitPack publication, GitHub releases, a signed local verification APK, Google Play AAB handoff, Zapstore publish, and post-release `-SNAPSHOT` bumps.

**Architecture:** Two sequential tagged releases on `master`. The library tag publishes via `library-release.yml` (JitPack + GitHub release). The app tag publishes via `app-release.yml` (signed APK + AAB + GitHub release) and is gated by a CI guard that rejects `-SNAPSHOT` library versions. Distribution to Google Play is manual (user uploads AAB); Zapstore is semi-manual (user runs `zsp publish` locally with their nsec).

**Tech Stack:** Gradle 8.x, Kotlin, Android Gradle Plugin, GitHub Actions (`library-release.yml`, `app-release.yml`), JitPack, `gh` CLI, `zsp` (Zapstore publisher).

**Spec:** `docs/superpowers/specs/2026-04-11-release-v1.9.0-v1.4.0-design.md`

**Starting state (verify before Task 1):**
- On branch `master`, working tree clean, up-to-date with `origin/master`

- `gradle.properties`: `appVersionName=1.3.3-SNAPSHOT`, `appVersionCode=133`
- Previous tags: `v1.8.3` and `app-v1.3.2`

## Rollback playbook

If something goes wrong mid-plan:

| Point of failure | Recovery |
|---|---|
| Before Task 3 push | `git reset --hard origin/master` — local commit is throwaway |
| Library CI red after Task 3 | Investigate logs (`gh run view --log`). Usually fix forward with a new commit + move the tag. Do **not** delete the tag if JitPack already cached a build. |
| JitPack 404 persists (Task 4) | Check `https://jitpack.io/com/github/davotoula/LightCompressor-enhanced/1.9.0/build.log`. Common causes: Gradle wrapper version, missing `maven-publish` output. Fix on master, push, re-tag as `v1.9.0` (force-update after deleting the bad tag with `git push --delete origin v1.9.0 && git tag -d v1.9.0`) — coordinate with user before force ops. |
| Wrong version in on-device label (Task 7) | Stop. Inspect `git show HEAD -- gradle.properties lightcompressor/gradle.properties`. Fix the bad file, amend or new commit, rebuild APK, re-verify. Do not proceed to Task 8. |
| App CI red after Task 8 | Inspect logs. If guard failed, `lightcompressor/gradle.properties` at the tagged commit contained `-SNAPSHOT` — means Task 6 was wrong. Fix with a follow-up commit, move the tag, force-push tag (coordinate with user). |
| Zapstore publish fails | Non-blocking: GitHub release is live. Diagnose with the error from `zsp`. Common issues: stale nsec, network, APK asset URL 404 (rare). |
| Play upload rejected | Non-blocking: user handles via Play Console. Likely cause: `versionCode=140` collides with a previous internal upload (should not — previous was `132`, then `133` was never uploaded). |
